# PowerShell OpenSSH Community Call Notes - May 21st, 2026

## Meeting notes

**PowerShell 7 Default Inclusion in Windows Server:** Artem Pronichkin from the Windows Server team announced that PowerShell 7 will be installed by default in the next version of Windows Server, discussed ongoing work to address compatibility issues, and invited community feedback, with Jason Helmick and Steve Lee highlighting the significance and next steps for migration and feedback channels.
    - **Announcement of Default Inclusion:** Artem Pronichkin shared that the next version of Windows Server (vNext) will include PowerShell 7 as a pre-installed application, making it immediately available after installation or upgrade, similar to how PowerShell 5.1 is currently provided.
    - **Compatibility and Work in Progress:** Artem explained that while the integration is ongoing, there are known incompatibilities between PowerShell 7 and certain Windows Server scenarios, and the team is actively collaborating with other Microsoft teams to resolve these issues before the final release.
    - **Insider Preview and Community Feedback:** Artem mentioned that a Windows Server Insider build previewing this functionality will be released in the near future, and encouraged the community to provide feedback via the Windows Server Insiders forum to help shape the integration and address any limitations.
    - **Migration Planning and Discussion Channels:** Steve Lee and Jason Helmick discussed the need for a smooth migration path from PowerShell 5.1 to 7, with plans to create a dedicated discussion in the PowerShell repository for community concerns and feedback regarding migration and compatibility.
- **DSC Version 3.2 and 3.3 Preview Updates:** Steve Lee provided updates on the release of DSC version 3.2 and its servicing release, highlighted a new registry list resource in the 3.3 preview, and encouraged feedback from users, with Jason Helmick facilitating the discussion.
    - **DSC 3.2 General Availability and Servicing Release:** Steve Lee announced that DSC 3.2 reached general availability three weeks prior, and a servicing release (3.2.1) was issued to fix an issue affecting PowerShell-based DSC resources located in paths with spaces.
    - **Registry List Resource in 3.3 Preview:** Steve introduced a new registry list resource in the 3.3 preview, allowing users to manage multiple registry keys and values as an array within a single resource, improving configuration simplicity and performance.
    - **Feedback and Community Involvement:** Steve encouraged users to test the new registry list resource and provide feedback in the DSC repository to help refine the feature.
- **PSResourceGet 1.3 Preview and Roadmap:** Anam Navied detailed the release of PSResourceGet 1.3 preview one, including new features like default registration of Microsoft Artifact Registry, concurrent installations, and bug fixes, and shared the team's roadmap and plans for future support, with Aditya Patwardhan and Sean Wheeler contributing technical details and documentation updates.
    - **Default Microsoft Artifact Registry Registration:** Anam explained that Microsoft Artifact Registry (MAR) is now registered as a default trusted repository for new users, with PS Gallery remaining as an untrusted repository, and provided instructions for existing users to manually register MAR.
    - **Concurrent Installations and Performance:** The release introduces concurrent installations for packages with dependencies, improving installation performance, and users are encouraged to provide feedback on this enhancement.
    - **DSC Resource Support in PSResourceGet:** Aditya Patwardhan described the addition of DSC resources for PSResourceGet, including repository configuration and module installation resources, which are implemented in PowerShell and available with the latest DSC version.
    - **Bug Fixes and Community Contributions:** Anam highlighted community pull requests that improved detection of PowerShell session types and enhanced pre-release label handling during updates, crediting contributors for their work.
    - **Roadmap and Future Plans:** Anam shared a blog post outlining the roadmap for PSResourceGet 1.3, including plans to standardize ORAS support, extend container registry compatibility, and drop support for Windows PowerShell in future releases, while maintaining support in earlier versions.
    - **Documentation Updates:** Sean Wheeler updated documentation to reflect the new repository registration process and changes in default repository priorities, providing guidance on managing repository settings.
- **PowerShell Gallery Migration and Reliability Improvements:** Amber Erickson discussed the ongoing migration of the PowerShell Gallery from cloud services to Azure Kubernetes Service to address scalability and reliability issues, described current search problems, and outlined upcoming security enhancements with Entre integration.
    - **Migration to Azure Kubernetes Service:** Amber explained that the PowerShell Gallery is being migrated from deprecated cloud services to Azure Kubernetes Service (AKS), which will provide a more scalable, container-based platform and facilitate modernization.
    - **Current Search and Indexing Issues:** Amber described ongoing intermittent search issues caused by limitations in the Lucene-based indexing system under heavy load, and noted that these are expected to improve after the migration to AKS.
    - **Workarounds and Status Updates:** Users experiencing search issues are advised to use PSResourceGet for searching and downloading packages, though some scenarios remain affected; updates are provided on the gallery status page.
    - **Entre Integration for Security:** Amber announced near-completion of Entre integration, which will replace long-lived API keys with identity-based authentication using short-lived tokens, aligning with security best practices and reducing credential risks.
- **Platform Support Updates for PowerShell Releases:** Anam Navied provided updates on PowerShell support for new Linux distributions, including RHEL 10 and Debian 13, as well as upcoming ARM and Ubuntu 26.04 support, with Jason Helmick summarizing the progress and future announcements.
    - **RHEL 10 and Debian 13 Support:** Anam announced that PowerShell 7.7 Preview 1 now supports RHEL 10 and Debian 13, with support for these distributions also planned for the upcoming 7.6 release.
    - **ARM and Ubuntu 26.04 Plans:** Support for Debian 12 and 13 ARM architectures is planned for future 7.6 releases, and work is underway to add Ubuntu 26.04 support, with more updates expected in the next community call.
- **MSI Deprecation and MSIX Transition:** Jason Helmick addressed the deprecation of MSI installers starting with PowerShell 7.7, explained ongoing efforts to address MSIX gaps based on community feedback, and outlined plans for transparent tracking and discussion of these issues.
    - **MSI Deprecation Announcement:** Jason clarified that MSI installers will be deprecated starting with PowerShell 7.7, while PowerShell 7.6 LTS will continue to support MSI for current deployments.
    - **Community Feedback and MSIX Gaps:** The team has received extensive feedback about MSIX limitations, and is actively working with other teams to address these gaps, with plans to publicly track and mark progress on GitHub.
    - **Ongoing Discussions and Transparency:** Jason committed to increased transparency by creating public lists of MSIX and Windows Server scenario gaps, and encouraged continued community input through discussion threads.
- **Mac Notarization and Installation Improvements:** Jason Helmick and Sean Wheeler announced the successful implementation of Mac notarization for PowerShell releases 7.4 through 7.7, described improvements to the installation process, and noted ongoing discussions about Brew packaging.
    - **Mac Notarization Implementation:** Jason announced that Mac notarization is now in place for PowerShell versions 7.4, 7.5, 7.6, and 7.7, following significant technical challenges in aligning Microsoft and Apple security requirements.
    - **Installation Process Enhancements:** Sean explained that notarized PKG files will allow users to install PowerShell on Mac by simply double-clicking the package, streamlining the installation experience.
    - **Brew Packaging Status:** While new Brew packaging is not yet committed, the team is monitoring the situation and will provide updates as they continue discussions with the Brew community.
- **Documentation and Community Contributions:** Sean Wheeler provided updates on recent documentation improvements, highlighted significant community contributions such as the new error handling article, and described updates related to recent PowerShell releases and features.
    - **Error Handling Documentation:** Sean credited a community contributor for compiling and clarifying PowerShell error handling terminology and behaviors, resulting in a comprehensive new documentation article.
    - **Release and Feature Documentation:** Sean updated documentation to cover new features in PowerShell 7.7, including fixes for switch parameter handling and the introduction of the PS application output encoding preference variable.
    - **Script Analyzer and Repository Registration:** Documentation was also updated for Script Analyzer 1.25 and for the new process of registering Microsoft Artifact Registry as a repository in PSResourceGet.
- **Upcoming Demos and Community Events:** Jason Helmick announced the postponement of the Plaster 2.0 demo to the next meeting, invited additional demo proposals, and previewed upcoming community events and conference updates.
    - **Demo Scheduling:** The Plaster 2.0 demo was postponed to the next community call due to a packed agenda, and Jason invited community members to propose additional demos for future meetings.
    - **Community Event Updates:** Jason recapped the recent PowerShell Summit North America and previewed the upcoming PowerShell Conference EU, with plans to provide updates from these events in future calls.
